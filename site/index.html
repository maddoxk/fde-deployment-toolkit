fdetk — one-command deploy, verify & observe

A forward-deployed engineer’s onboarding toolkit. Scaffold a customer service from parameters, deploy it, run health & smoke checks against the running service, and emit a live status page — all from one zero-dependency Go CLI.

Live status pageRendered from real check runs, not hand-faked

What it does

Each subcommand emits an artifact rather than a Slack message, so onboarding becomes reproducible and auditable:

CommandActionOutput
scaffoldRender config + k8s manifest from customer paramsservice.config.json, deploy.yaml
deployStart (simulate deploying) the sample service locallyrunning service + endpoints
healthcheckLiveness / readiness / metrics probes (gates CI)3 check results, non-zero exit on failure
smokeAPI echo round-trip, content-type, metrics-advance3 check results
statusRun all checks & emit observability pagestatus.html + status.json

Quick start

git clone https://github.com/maddoxk/fde-deployment-toolkit
cd fde-deployment-toolkit
go build -o fdetk ./cmd/fdetk

./fdetk scaffold --customer "Northwind Logistics" --tier enterprise --replicas 4
./fdetk deploy    --customer "Northwind Logistics"
./fdetk healthcheck --customer "Northwind Logistics"
./fdetk smoke       --customer "Northwind Logistics"
./fdetk status      --customer "Northwind Logistics" --out ./site

The live status page

The status page on this site was produced by actually running fdetk status against a freshly started sample service for Northwind Logistics (production / eu-west-1 / enterprise tier). It shows uptime, request volume, p50/p95/p99 latency, error rate, request & latency sparklines, and the full table of 6 check results.
